# Quality Scorecard (Evaluation)
## PEARL Alignment Document

This document defines the quality gates for the **Quality Scorecard** UI interface (`QualityScorecard.jsx`), accessible via the Yardmaster Toolkit. It follows the **EYE** (Evaluation, Yield, Evolve) stage of the ADDIECRAPEYE lifecycle.

### P - Perspective
*Who is the user in this view, and what is their immediate goal?*
- **User State (Cognitive Load: High - Analytical):** The user acts as an "Evaluator / QA Auditor". They are shifting from content-creation into strict academic analysis.
- **Goal:** To paste text or submit a RAG document ID to receive a 5-dimension pedagogical grade (Bloom's, ADDIE, Accessibility, Engagement, Assessment), complete with concrete recommendations for improvement.

### E - Engineering
*What backend services, APIs, or components drive this view?*
- **Component:** `QualityScorecard.jsx` provides a dedicated view for the evaluation framework.
- **Backend Bindings (Rust):**
  - **`crate::quality_scorecard::evaluate_document`**: Hits `/api/yard/score` to run text against the deeply typed `PedagogicalScore` JSON schema.
  - **`crate::journal::export_analytics`**: Includes a direct webhook trigger to export telemetry data to a Canvas LMS format (`/api/analytics/export`).
- **Dependencies:** 
  - Submits dual-mode input: raw text (`paste`) or vector database retrieval (`rag`).

### A - Aesthetic
*What is the visual language, color scheme, and mood?*
- **Palette/Mood:** Academic Dashboard. It relies heavily on color-coding to convey quality at a glance (A = Neon Green `#4ec9b0`, C = Gold `#cfb991`, F = Red `#f44747`).
- **Key Elements:** 
  - The `scorecard-grade-circle` creates a glowing focal point indicating the overall letter grade and percentage.
  - The `ScoreBar` components use a `linear-gradient` filling animation to visualize the 5 specific dimensions, making the subjective AI evaluation feel like an objective, quantitative measurement.

### R - Research
*What data informs this design? How does it align with the Fancy Bible?*
- **Objective Grading Structure:** This fulfills the "Evaluation" phase of the ADDIE model perfectly. Rather than asking Pete (a conversational agent in `crate::agent`) "is this good?", the Yardmaster uses `crate::quality_scorecard` to parse the text against a strict JSON schema, enforcing that ID best practices are explicitly measured rather than implicitly assumed.

### L - Layout (C.R.A.P. Methodology)
*How does this view use Contrast, Repetition, Alignment, and Proximity?*
- **Contrast:** Heavily reliant on academic color boundaries. The specific letter grades (A = Neon Green `#4ec9b0`, C = Gold `#cfb991`, F = Red `#f44747`) contrast sharply with the dark slate text to instantly convey failure or success.
- **Repetition:** The five Dimension Bars repeatedly use the exact same `linear-gradient` filling animation, teaching the user that all 5 criteria (Bloom's, Accessibility, etc) are weighted equally.
- **Alignment:** Results split horizontally on wide screens (Grade Circle left, Details right) to maximize data density and compare overall score directly against granular dimension breakdowns.
- **Proximity:** The actionable recommendation text sits intimately beneath the specific dimension score bar it references. The feedback is never detached from the score it explains.
