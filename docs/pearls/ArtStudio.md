# Art Studio (Creative Suite)
## PEARL Alignment Document

This document defines the quality gates for the **Art Studio** UI interface (`ArtStudio.jsx`), which manifests the "AI · Fun" operating mode. It follows the **EYE** (Evaluation, Yield, Evolve) stage of the ADDIECRAPEYE lifecycle.

### P - Perspective
*Who is the user in this view, and what is their immediate goal?*
- **User State (Cognitive Load: Low - Conversational):** The user is a "High School Gym Coach." They do not know what CFG scale, denoising strength, or seeds are, and they hate computers. They are here to converse with the AI to magically spawn media in a highly intuitive, frictionless space.
- **Goal:** To chat with Programmer Pete and ask him to build assets (images, music, 3D models), instantly previewing those spatial payloads inside the standalone Daydream viewport without leaving the application.

### E - Engineering
*What backend services, APIs, or components drive this view?*
- **Component:** `ArtStudio.jsx` (Redesigned with Premium Glassmorphic aesthetic and tested via `Vitest`).
- **Backend Bindings (Rust):**
  - **`crate::trinity_api::trinity_chat`**: The unified inference router that drives the conversational stream in the left pane.
  - **`crate::vaam_bridge::*`**: Intercepts chat messages and scans for vocabulary usage, rewarding Coal metrics.
  - **`crate::creative::get_image` & `get_audio`**: Exposes proxy generation endpoints to ComfyUI/MusicGPT.
  - **`crate::narrative::narrator_stream`**: Translates simple generation requests into Great Recycler dynamic prose responses.
  - **`crate::health::check_sidecar_health`**: Real-time SSE polling for `StatusBadge` neon activity indicators.
- **Dependencies:** 
  - Tauri native process IPC bound connects the UI to the standalone Native Bevy `art_studio` OS process.
  - RAG index contextualization connects historical user workflows to the generated art outputs.

### A - Aesthetic
*What is the visual language, color scheme, and mood?*
- **Palette/Mood:** "The Window to the Imagination". It is a clean, hyper-modern Media Suite. All "hacker" elements and raw JSON logs have been banished to the Yardmaster. Badges for sidecars glow neon green (`#34d399`) when active.
- **Key Elements:** 
  - `media-chat` replacing technical forms. The user simply types instructions natively into a chat bar.
  - `AssetCard` provides responsive image/video thumbnails and audio players directly in the directory.

### R - Research
*What data informs this design? How does it align with the Fancy Bible?*
- **Pedagogical Alignment:** Anchors the "Aesthetic" and "Yield" steps of the ADDIECRAPEYE lifecycle. The "Creative Flow" relies on low-friction conversational chat (`media-chat`) instead of high-cognitive-load dropdowns.
- **Isomorphism:** The Holy Trinity logic pipeline connects `crate::quests` (content parsing), `crate::trinity_api` (Socratic guide), and `crate::creative` (asset generation) directly to the right-pane Daydream scene graph synchronization `sync-scene`.
- **Hardware Limitations:** The Strix Halo NPU offloads whisper/TTS leaving the 120GB GPU fully saturated for the simultaneous Mistral context windows and ComfyUI generation bursts.

### L - Layout (C.R.A.P. Methodology)
*How does this view use Contrast, Repetition, Alignment, and Proximity?*
- **Contrast:** The Daydream Canvas commands the absolute highest visual contrast and pixel real estate. The conversational chat interface drops into the `#0F0D0A` background to avoid stealing focus from the rendered 3D world.
- **Repetition:** Pete's conversational responses utilize the exact same structural padding as the `PhaseWorkspace.md` chat, ensuring the user immediately recognizes that they are still talking to their Co-Pilot.
- **Alignment:** Strictly adopts a permanent split-pane layout to protect the media rendering zone. The left pane (30%) is dedicated entirely to the Chat Thread and Asset Gallery. The right pane (70%) is entirely consumed by the Daydream Viewport.
- **Proximity:** The chat input field acts as the anchor point of the left column, physically grouped tightly with the active asset thumbnails, enforcing that "conversation == creation".
