# Character Sheet (Identity)
## PEARL Alignment Document

This document defines the quality gates for the **Character Sheet** UI interface (`CharacterSheet.jsx`), accessible via the Portfolio tab. It follows the **EYE** (Evaluation, Yield, Evolve) stage of the ADDIECRAPEYE lifecycle.

### P - Perspective
*Who is the user in this view, and what is their immediate goal?*
- **User State (Cognitive Load: Low):** The user acts as a reflective "Student" reviewing their performance. They are assessing overarching accomplishments rather than performing immediate, stressful generative tasks.
- **Goal:** To verify their meta-game progression (Coal, Steam, Resonance), trace their mastery of the IBSTPI skill domains, and confidently assess their timeline towards graduation and the final Portfolio Yield.

### E - Engineering
*What backend services, APIs, or components drive this view?*
- **Component:** `CharacterSheet.jsx` is the primary data-aggregation dashboard.
- **Backend Bindings (Rust):**
  - **`crate::character_sheet::get_character`**: Parses the `Character` SQLite row detailing the user's meta-game progression, IBSTPI skill domains, and Resonance levels.
  - **`crate::quests::get_active_quest`**: Returns the `IronRoadState` containing which `AddiecrapeyePhase` the user is currently tackling.
  - **`crate::journal::get_completed_phases`**: Used to compute graduation readiness via milestone snapshots.
- **Dependencies:** 
  - Resolves `Promise.all` across `/api/character`, `/api/quest`, and `/api/pearl` on mount.
  - Dynamically computes the `overallMaturation` percentage based on the alignment of `completedPhases` against 5 distinct maturation dimensions (Content, Production, Pedagogy, Design, Reflection).

### A - Aesthetic
*What is the visual language, color scheme, and mood?*
- **Palette/Mood:** Esoteric RPG Character Menu. It leans heavily into "glassmorphism" (`backdrop-filter: blur(12px)`) over the dark void. Golden (`#CFB991`), Neon Blue (`#22d3ee`), and Violet (`#a78bfa`) accents demark the ADDIE, CRAP, and EYE sections respectively.
- **Key Elements:** 
  - The "Hook Book" banner serves as a highly-styled animated callout tying the user back to "The Four Chariots".
  - The 12 Station Cards explicitly map Bloom's Taxonomy, Hero's Journey Chapters, and the PEARL lens focused on that phase.
  - Progress bars use glowing gradients (`linear-gradient(90deg, #ea580c, #fb923c)` for Coal) to reinforce the steam-engine metaphor.

### R - Research
*What data informs this design? How does it align with the Fancy Bible?*
- **Pedagogical Concrete Mapping:** This view is the definitive realization of the "Trinity Identity" (ID). It grounds the abstract AI workflows managed by `crate::agent` into concrete Instructional Design metrics tracked in `character_sheet.rs` (QM Score, IBSTPI, ATD, AECT).
- **Proof of Work:** It proves to the user that they are not just generating random assets, but systematically fulfilling a rigid pedagogical contract enforced by `crate::quality_scorecard`.

### L - Layout (C.R.A.P. Methodology)
*How does this view use Contrast, Repetition, Alignment, and Proximity?*
- **Contrast:** Glowing gradient progress bars (`#ea580c` to `#fb923c`) act as explosive focal points against the vast, translucent glassmorphic voids, prioritizing "Progress Visualization" over static text. 
- **Repetition:** The visual language of the 12 Station cards iterates identically (Icon + Phase Name + Descriptor), forcing the user to recognize ADDIECRAPEYE as a rhythmic pedagogical loop rather than a set of disconnected tools.
- **Alignment:** Data density dictates a strict `max-width: 1200px` container centering the dashboard in the viewport. This protects the eye from dragging infinitely across ultra-wide monitors when scanning dense tracking tables.
- **Proximity:** The specific "Gate Review" artifact requirements are housed immediately next to the primary "Graduation Submission" mechanism. The user never searches for what is missing; the deficit is adjacent to the goal.
