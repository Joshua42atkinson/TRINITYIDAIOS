# Phase Workspace (Iron Road)
## PEARL Alignment Document

This document defines the quality gates for the **Phase Workspace** UI interface (`PhaseWorkspace.jsx`), taking place on the "Iron Road". It follows the **EYE** (Evaluation, Yield, Evolve) stage of the ADDIECRAPEYE lifecycle.

### P - Perspective
*Who is the user in this view, and what is their immediate goal?*
- **User State (Cognitive Load: Medium - Conversational):** The user is "riding the Iron Road" as a student/designer. They are in the "Inhale" phase—planning, arguing, and learning.
- **Goal:** To converse with "The Great Recycler" (Socratic AI), tame emerging "Scope Creep", unlock vocabulary via VAAM, and clear objectives to advance ADDIECRAPEYE stages.

### E - Engineering (Rust Backend Mapping)
*What backend services, APIs, or components drive this view?*
- **Component:** `PhaseWorkspace.jsx` is the heart of the guided experience, parsing complex JSON SSE events to render animated transitions.
- **Backend Bindings (Rust):**
  - **`crate::quests` (`/api/game/...`)**: This module strictly defines the ADDIECRAPEYE phases. Endpoints like `get_game_state` return the calculated "Product Maturity" and "XP" metrics. `advance_phase` triggers automatic background git commits and chronological `journal.rs` snapshots.
  - **`crate::project` & `persistence.rs`**: The overall `GameState` is managed under `AppState.project` and periodically flushed to `trinity_projects` tables in SQLite to guarantee quest progression survives reboots.
  - **`crate::agent` (`/api/agent/chat`)**: The PhaseWorkspace hooks into the main chat loop. Unlike the Yardmaster, this view enforces `track_mechanics = true` and `enforce_narrative = true`, physically restricting the user to the "Great Recycler" Persona.
  - **`crate::cow_catcher` (Autopoiesis)**: Monitors the dialogue for divergence. If the conversation strays too far from the current phase objective, the system triggers the "Scope Creep" interruption workflow.

### A - Aesthetic
*What is the visual language, color scheme, and mood?*
- **Palette/Mood:** High-fantasy meets industrial steam. Phase colors coordinate with Bloom's Taxonomy (e.g., Gold for Remember/Analyze, Blue for Understand/Apply). The UI relies on glowing orbs and ceremonial transition banners (`phase-transition`).
- **Key Elements:** 
  - The Socratic voice speaks in formatted Markdown (the "Book Stage").
  - The Scope Creep modal uses a color-coded threat element glow.
  - `handoff-banner` explicitly signals the transition from "Vision set" (Great Recycler) to "Ready to build" (Programmer Pete), rendering a gradient button.

### R - Research
*What data informs this design? How does it align with the Fancy Bible?*
- **Iron Road Duality:** Strongly enforces the "Inhale" vs "Exhale" duality outlined in `TRINITY_FANCY_BIBLE.md`. The `conductor_leader.rs` restricts the user to *planning and reflection* by injecting "The Great Recycler" preambles into `agent.rs`.
- **Effortless Effort:** Once objectives are cleared, the handoff banner explicitly instructs the user to open Pete's Yardmaster Terminal to execute the code. This explicitly separates planning (frontend) from execution (backend code generation).

### L - Layout (C.R.A.P. Methodology)
*How does this view use Contrast, Repetition, Alignment, and Proximity?*
- **Contrast:** The AI's Socratic output (The "Book Stage") is rendered in large, elegant serif fonts, contrasting aggressively with the user's small, utilitarian text inputs. Unmet objectives pulse with a high-contrast glow until cleared.
- **Repetition:** Systemic threats (like "Scope Creep") always spawn via the exact same red-shifted modal transition, reflexively conditioning the user to recognize when they stray off-topic.
- **Alignment:** The primary narrative scroll is single-column and center-weighted with a strict `max-width`, preventing long text lines from stretching across ultra-wide monitors and fatiguing the reader. The Objectives bar locks to the top.
- **Proximity:** The `MicButton` physically touches the prompt `<textarea>`, implicitly bounding the two primary interaction paradigms (Voice vs. Text) into a single indisputable input zone.
