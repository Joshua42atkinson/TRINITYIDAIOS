# Journal Viewer (Reflection)
## PEARL Alignment Document

This document defines the quality gates for the **Journal Viewer** UI interface (`JournalViewer.jsx`). It follows the **EYE** (Evaluation, Yield, Evolve) stage of the ADDIECRAPEYE lifecycle.

### P - Perspective
*Who is the user in this view, and what is their immediate goal?*
- **User State (Cognitive Load: High - Introspective):** The user acts as a "Reflective Practitioner". They are pausing the external action to document their internal metacognitive state.
- **Goal:** To view an immutable timeline of their project's history, log manual "Weekly Reflections", and export these snapshots as raw HTML documents for grading.

### E - Engineering
*What backend services, APIs, or components drive this view?*
- **Component:** `JournalViewer.jsx` maps an array of `journal-entry` objects into an expandable accordion timeline.
- **Backend Bindings (Rust):**
  - **`crate::journal::get_timeline`**: Fetches the active history of chapter snapshots via `GET /api/journal`.
  - **`crate::journal::add_reflection`**: Creates new user-authored reflections tied to the current phase via `POST /api/journal`.
  - **`crate::journal::export_snapshot`**: Provides the HTML generation logic at `/api/journal/export/{id}` for external grading.
- **Dependencies:** 
  - Resolves timeline updates automatically upon advancing phases in `crate::quests`.

### A - Aesthetic
*What is the visual language, color scheme, and mood?*
- **Palette/Mood:** Clean, chronological, and archival. Less "gamified" than the main Iron Road, leaning towards a structured data-table/timeline format. 
- **Key Elements:** 
  - Standardized emoji typologies (🚉 for Phase, 🏆 for Chapter, 📓 for Reflection) provide immediate visual scanning of event types.
  - The accordion expansion (`expanded` state) reveals underlying game mechanics at the exact moment of the snapshot (e.g., "Phase: Contrast", "XP: 340", "Resonance: 12").

### R - Research
*What data informs this design? How does it align with the Fancy Bible?*
- **Evaluation Mechanics:** Focuses heavily on the "Evaluation" (E in ADDIE) and "Perspective/Research" (P/R in EYE) phases. The system believes that true learning occurs in the reflection of the action, not just the action itself.
- **Artifact Persistence:** Backed by `crate::journal`, the viewer guarantees that ephemeral chat (from `crate::agent`) is systematically solidified into immutable SQLite records.

### L - Layout (C.R.A.P. Methodology)
*How does this view use Contrast, Repetition, Alignment, and Proximity?*
- **Contrast:** The vertical chronological spine acts as a stark, high-contrast line. When an accordion node expands, it leverages a lighter slate background to distinctively separate its contents from the surrounding dark void.
- **Repetition:** Standardized emoji typologies (🚉 for Phase, 🏆 for Chapter, 📓 for Reflection) repeat identically on every node, training the user's eye to filter timeline events rapidly by color/shape.
- **Alignment:** Strictly center-weighted to a single vertical spine. The `<textarea>` for authoring new reflections aligns perfectly flush with previous historical text outputs.
- **Proximity:** The metadata associated with a snapshot (e.g., "Phase: Contrast", "XP: 340") is housed intimately inside the accordion fold directly above the reflection text, ensuring the context is physically bound to the thought.
