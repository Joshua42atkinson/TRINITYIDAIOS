# The Hook Book
## PEARL Alignment Document

This document defines the quality gates for the **Hook Book** native UI interface within the `CharacterSheet.jsx` view. It follows the **EYE** (Evaluation, Yield, Evolve) stage of the ADDIECRAPEYE lifecycle.

### P - Perspective
*Who is the user in this view, and what is their immediate goal?*
- **User State (Cognitive Load: Low/Exploratory):** The user acts as a reflective "Student" or an exploratory "Architect". They are not currently in the stressful Forge (Development) phase but are taking stock of the systemic capabilities.
- **Goal:** To understand *what* Trinity can do. The user needs to see the 37 Hooks divided into the 4 Schools (Pedagogy, Creation, Systems, Identity) and understand how they can be chained together to conquer the Iron Road.

### E - Engineering
*What backend services, APIs, or components drive this view?*
- **Component:** A native sub-view inside the `CharacterSheet.jsx` component. Uses a Tab system (`activeTab === 'hookbook'`).
- **Data Source:** It dynamically fetches `HOOK_BOOK.md` from the root workspace via the frontend public `/docs/HOOK_BOOK.md` or a direct fetch, parsing the Markdown using `marked`.
- **System Integration:** By making the Hook Book a first-class feature inside the Character Sheet, it reinforces the Isomorphic principle—academic knowledge mapping directly to game mechanics.

*What is the visual language, color scheme, and mood?*

*What data informs this design? How does it align with the Fancy Bible?*
- **Pedagogical Concrete Mapping:** According to the Trinity Fancy Bible, learning is scaffolded. The Hook Book provides the necessary "Worked Examples" and "Advance Organizers".

### L - Layout (C.R.A.P. Methodology)
*How does this view use Contrast, Repetition, Alignment, and Proximity?*
- **Contrast:** Glowing badges for readiness Tiers (🟢 Cast, 🟡 Scribed, 🔴 Prophesied) contrast sharply with the dark background to instantly communicate feature completeness.
- **Repetition:** The structural breakdown of the 4 Schools (Pedagogy, Creation, Systems, Identity) leverages standard Markdown tables, rendered cleanly by Github-Flavored Markdown.
- **Alignment:** Clean, single-column reading mode within the 1200px max-width container keeps cognitive drag minimal while absorbing deep systemic knowledge.
- **Proximity:** By moving the Hook Book *inside* the Character Sheet, the tools (Hooks) are placed directly beside the Progression (Contract) and the Context (Player Identity).
