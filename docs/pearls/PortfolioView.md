# Portfolio View (Graduation)
## PEARL Alignment Document

This document defines the quality gates for the **Portfolio View** UI interface (`PortfolioView.jsx`), the ultimate output deliverable of the Trinity system. It follows the **EYE** (Evaluation, Yield, Evolve) stage of the ADDIECRAPEYE lifecycle.

### P - Perspective
*Who is the user in this view, and what is their immediate goal?*
- **User State (Cognitive Load: Low - Passive Pride):** The user has officially stepped out of the "Student/Player" role and assumed the posture of a "Graduated Professional."
- **Goal:** To view, filter, and eventually export a verified, competency-aligned professional portfolio demonstrating mastery over instructional design standards.

### E - Engineering
*What backend services, APIs, or components drive this view?*
- **Component:** `PortfolioView.jsx` acts as a static site generator *within* the React app.
- **Backend Bindings (Rust):**
  - **`crate::journal::export_portfolio`**: The orchestration endpoint (`GET /api/eye/portfolio`) that joins the `Journal` snapshots with the user's `Character` table to hydrate this view.
  - **`crate::quality_scorecard::compute_scores`**: Provides the deeply typed `competencyData` structures (QM Score, IBSTPI) injected into the payload.
- **Dependencies:** 
  - Sub-tabs (`Overview`, `Evidence`, `Badges`) manage state locally based on the loaded JSON.
  - Dynamically injects a CSS stylesheet (`portfolio-view-styles`) on mount to ensure its unique keyframe animations run independently of the global OS theme.

### A - Aesthetic
*What is the visual language, color scheme, and mood?*
- **Palette/Mood:** "Museum Quality." It abandons the utilitarian terminal vibes of the Yardmaster in favor of premium presentation features. Heavy use of the `Cinzel` serif font, gold (`#CFB991`) text-shadows, and pulsing gradients. 
- **Key Elements:** 
  - The Header pulses with a 6-second `pv-shimmer` animation across gold gradient text to instantly convey "Premium Deliverable."
  - Badges expand securely inline, utilizing `linear-gradient(145deg)` to create depth against the dark background.

### R - Research
*What data informs this design? How does it align with the Fancy Bible?*
- **The Yield Phase:** This is the literal "Yield" phase of EYE. The system promises that Trinity delivers three things: the OS (code), the AI (novel), and the ID (Portfolio).
- **Rigor Validation:** This view validates the pedagogical rigor of `crate::quality_scorecard` by proving to the user that every conversational action taken via `crate::agent` explicitly maps to an internationally recognized competency.

### L - Layout (C.R.A.P. Methodology)
*How does this view use Contrast, Repetition, Alignment, and Proximity?*
- **Contrast:** Abandons the utilitarian dark terminal vibes entirely. Introduces a bright "Museum Quality" aesthetic leveraging negative space and aggressive gold keyframe animations (`pv-shimmer`) across typography to scream "Premium Reward."
- **Repetition:** The IBSTPI badges repeat their core geometric shield shapes constantly, projecting a sense of rigid, unyielding systemic standard across different skill domains.
- **Alignment:** Relies on dense, horizontally flush filter rows (`pv-filter`) followed by expansive masonry grids. It behaves explicitly like a modern showcase website (e.g., Awwwards or Behance) rather than a software dashboard.
- **Proximity:** Extracted evidence artifacts (e.g., generated lesson plans) physically touch the pedagogical competency tag they are meant to prove. The system never visually separates the 'Claim' from the 'Proof'.
