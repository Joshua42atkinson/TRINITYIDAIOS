# Chariot Viewer (Documentation)
## PEARL Alignment Document

This document defines the quality gates for the **Chariot Viewer** UI interface (`ChariotViewer.jsx`), the in-app markdown reader for core system lore. It follows the **EYE** (Evaluation, Yield, Evolve) stage of the ADDIECRAPEYE lifecycle.

### P - Perspective
*Who is the user in this view, and what is their immediate goal?*
- **User State (Cognitive Load: Low):** The user is a "Seeker of Knowledge" stepping away from generation into passive absorption.
- **Goal:** To read deep, structured markdown documentation (the "Four Chariots" system lore) without the jarring context switch of exiting the immersive OS environment into an external browser.

### E - Engineering
*What backend services, APIs, or components drive this view?*
- **Component:** `ChariotViewer.jsx` is a functional React component wrapping the `marked` library.
- **Backend Bindings (Rust):**
  - **`crate::main::serve_docs`**: (or equivalent static file router) Serves the markdown assets directly from the workspace.
  - **`crate::inference_router` & `crate::agent::HotelGuard`**: While primarily a reading view, it is designed to eventually surface real-time retrieval matrices and telemetry logs from the inference router, showing exactly how documentation drives system prompts.
- **Dependencies:** 
  - Performs a direct `fetch()` to static assets in the `/docs/` directory.
  - Attaches a `scroll` event listener to the `contentRef` to calculate mathematical reading progress.
  - Provides a direct HTML5 `download` attribute link for exporting the raw `.md` file.

### A - Aesthetic
*What is the visual language, color scheme, and mood?*
- **Palette/Mood:** "The Library / Reader Mode." It aggressively strips away UI clutter in favor of typography. The background drops to absolute black (`#0F0D0A`), making the text the sole focus.
- **Key Elements:** 
  - A subtle 2px golden (`#CFB991`) progress bar tracks reading completion along the top border.
  - The `chariot-article` class applies distinct markdown typography rules (large serifs, spacious line-heights, indented blockquotes) to make long-form reading comfortable.

### R - Research
*What data informs this design? How does it align with the Fancy Bible?*
- **Contextual Anchoring:** This is the manifestation of the "Self-Documenting System." The Fancy Bible mandates that the system's own lore and operation manuals must be accessible natively.
- **Agentic Memory Transparency:** By integrating the Chariots into the UI, the user is constantly tethered to the "Why" and "How" of the ADDIECRAPEYE methodology. It aligns exactly with the preamble strategies generated by `crate::agent::inject_persona_preamble`, making the system's logic completely transparent to the user.

### L - Layout (C.R.A.P. Methodology)
*How does this view use Contrast, Repetition, Alignment, and Proximity?*
- **Contrast:** Aggressively strips away UI clutter in favor of typography. The background drops to absolute black (`#0F0D0A`), making the stark text the sole focus of the viewport.
- **Repetition:** Markdown styling (headers, `blockquote` indentation, list bullets) aggressively repeats standard publishing norms so the user intuitively knows how to scan the document.
- **Alignment:** Center-weighted, single-column reading flow (`max-width: 800px`) similar to Medium or Substack. By breaking out of the standard grid (`gridColumn: '1 / -1'`), it behaves as a full-screen, focused overlay.
- **Proximity:** Utilitarian actions like 'Download' and 'Back' are pinned to the extreme outer corners of the sticky header, intentionally pushed far away from the text body to keep them out of the reading flow.
